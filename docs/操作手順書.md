# スカイピア削除自動化ツール 操作手順書

## 1. このツールの目的

このプロジェクトは、楽天RMSと連携しているスカイピア管理ツール上で、商品データの削除作業を自動化するためのツールです。

Python と Playwright を使ってブラウザを自動操作し、以下の作業を行います。

- スカイピア管理ツールへのログイン
- 商品管理画面への移動
- 商品番号を指定した削除
- 削除結果のCSV記録
- エラー時のスクリーンショット保存

今後の実装対象として、商品管理「削除済み」の完全削除、商品番号指定削除から削除済み完全削除までの連続実行、出品選択の全削除があります。

## 2. とても重要な安全ルール

削除確認画面に「対象のASINをNG登録する」というチェック項目が表示される場合があります。

このツールでは、このチェックは絶対に入れません。もし最初からチェック済みになっている場合は、自動でチェックを外してから削除します。

このルールは、ASINをNG登録することが目的ではなく、商品データの削除だけを行うためのものです。

また、大量削除はリスクが高いため、初めて動かすときは必ず少数の商品番号で確認してください。最初は1件から2件程度で試すことを推奨します。

## 5. 初回セットアップ

### 5-1. プロジェクトフォルダへ移動

PowerShellで以下を実行します。

```powershell
cd "C:\Users\allja\Desktop\codex-project\スカイピア"
```

### 5-2. 必要なPythonパッケージを入れる

```powershell
pip install playwright python-dotenv pytest pytest-asyncio
python -m playwright install chromium
```

### 5-3. `.env` を作成する

`.env.example` を参考にして、同じフォルダに `.env` を作成します。

```text
SKYPIEA_LOGIN_EMAIL=ログインメールアドレス
SKYPIEA_LOGIN_PASSWORD=ログインパスワード
```

注意:

- `.env` はGitHubに上げません。
- パスワードをチャットやREADMEに書かないでください。
- `.env.example` には実ログイン情報を書かないでください。

## 6. ログイン確認

まず、削除処理を動かす前にログインと商品管理画面への移動だけを確認します。

```powershell
python src/main.py --mode login-check
```

成功すると、`output/delete_results.csv` に `login-check` の結果が記録されます。

失敗した場合は、`output/screenshots/` にエラー時の画面が保存されます。画面構成が変わっている場合は、`config.json` のセレクタ調整が必要です。

## 7. 商品番号指定削除の使い方

### 7-1. 商品番号を入力する

`input/item_numbers.txt` に、削除したい商品番号を1行ずつ入力します。

例:

```text
ITEM001
ITEM002
ITEM003
```

空行は無視されます。前後の空白も取り除かれます。同じ商品番号が重複している場合は、1回だけ処理されます。

### 7-2. 削除を実行する

```powershell
python src/main.py --mode delete-by-item-numbers
```

処理の流れは以下です。

1. スカイピア管理ツールにログインする。
2. 商品管理画面へ移動する。
3. `input/item_numbers.txt` から商品番号を読み込む。
4. 商品番号を一定件数ごとに分ける。
5. 商品管理画面で商品番号を検索する。
6. 検索結果がある場合、表示中の商品を選択する。
7. 選択データ削除を実行する。
8. 削除確認画面で「対象のASINをNG登録する」がチェックされていれば外す。
9. 削除を確定する。
10. 結果を `output/delete_results.csv` に記録する。

### 7-3. 最初の実行時の注意

初回は必ず1件から2件だけで試してください。

実画面のボタン名や入力欄が想定と違う場合、処理が途中で止まることがあります。その場合、スクリーンショットを見て `config.json` のセレクタを修正します。

## 8. 実行結果の確認

処理結果は `output/delete_results.csv` に記録されます。

列の意味は以下です。

| 列名 | 意味 |
|---|---|
| `datetime` | 記録日時 |
| `mode` | 実行モード |
| `loop` | 処理回数 |
| `display_count` | その回で処理した件数 |
| `deleted_count` | 削除できた件数 |
| `result` | 結果。`success`、`error`、`finished`、`no_target` など |
| `detail` | 詳細メッセージ |
| `screenshot` | エラー時スクリーンショットの保存先 |

通常ログは `output/logs/` に保存されます。

エラー時の画面は `output/screenshots/` に保存されます。

## 9. 代表的なエラーと対応

### `.env` のログイン情報がない

表示例:

```text
SKYPIEA_LOGIN_EMAIL and SKYPIEA_LOGIN_PASSWORD must be set in .env or environment.
```

対応:

`.env` を作成し、ログインメールとパスワードを入力してください。

### 商品管理画面に移動できない

原因:

- メニュー名やリンクが変わった。
- 商品管理画面のURLを直接指定したほうが安定する。

対応:

`config.json` の `product_management_url` に商品管理画面のURLを設定します。未設定の場合は、「商品管理」リンクをクリックして移動します。

### ボタンや入力欄が見つからない

原因:

- 実画面のHTMLやボタン名が想定と違う。

対応:

`output/screenshots/` のスクリーンショットを確認し、`config.json` の `selectors` を実画面に合わせて修正します。

### 削除対象が0件になる

原因:

- 商品番号が間違っている。
- すでに削除済み。
- 検索条件や入力欄の指定が実画面と合っていない。

対応:

まず手動で同じ商品番号を画面検索し、検索結果が出るか確認してください。

## 10. 現在実装済みのモード

| コマンド | 状態 | 内容 |
|---|---|---|
| `python src/main.py --mode login-check` | 実装済み | ログインして商品管理画面まで移動 |
| `python src/main.py --mode delete-by-item-numbers` | 実装済み | 商品番号指定削除 |
| `python src/main.py --mode delete-deleted-items` | 未実装 | 商品管理「削除済み」完全削除 |
| `python src/main.py --mode delete-by-item-numbers-and-clean` | 未実装 | 商品番号指定削除から削除済み完全削除まで連続実行 |
| `python src/main.py --mode delete-listing-candidates` | 未実装 | 出品選択 全削除 |

未実装モードは、今の段階では実行すると未実装エラーになります。

## 11. 設定ファイルの見方

`config.json` では主に以下を調整します。

| 設定 | 意味 |
|---|---|
| `login_url` | ログイン画面URL |
| `product_management_url` | 商品管理画面URL。空欄ならメニュークリックで移動 |
| `default_display_count` | 基本表示件数。仕様では200件 |
| `fallback_display_counts` | 200件で不安定な場合に下げる候補 |
| `disabled_display_counts` | 通常使わない表示件数 |
| `item_number_delete_chunk_size` | 商品番号指定削除で一度に検索する件数 |
| `wait_after_delete_seconds` | 削除後に待つ秒数 |
| `max_retry` | リトライ回数の設定値 |
| `page_load_timeout_seconds` | ページ読み込み待ち秒数 |
| `save_screenshot_on_error` | エラー時スクリーンショット保存の有無 |
| `headless` | ブラウザ画面を表示しないかどうか。初期は `false` |
| `selectors` | 画面上の入力欄やボタンを探すための候補 |

開発・確認中は `headless: false` のままにして、実際のブラウザ画面を見ながら確認するのが安全です。

## 12. 運用時のおすすめ手順

1. `.env` が正しく設定されているか確認する。
2. `python src/main.py --mode login-check` を実行する。
3. 商品管理画面まで移動できることを確認する。
4. `input/item_numbers.txt` に商品番号を1件から2件だけ入れる。
5. `python src/main.py --mode delete-by-item-numbers` を実行する。
6. `output/delete_results.csv` を確認する。
7. 問題がなければ、件数を少しずつ増やす。
8. エラーが出た場合は、`output/screenshots/` と `output/logs/` を確認する。
